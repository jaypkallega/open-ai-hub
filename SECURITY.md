# Security Policy

## Reporting Harmful Content

If you discover a prompt or skill file in this repository that could
be used for harmful purposes — including but not limited to:

- Jailbreak attempts
- Harmful instruction generation
- Privacy violations
- Social engineering scripts
- Content targeting minors

**Please DO NOT open a public Issue.**

Instead, use GitHub's private vulnerability reporting:
> Repository → Security tab → Report a vulnerability

All reports will be reviewed within 48 hours by a Core Maintainer.

## Scope
This policy covers all files in the `prompts/`, `skills/`,
`templates/`, `experimental/`, and `community-picks/` folders.

## Our Commitment
We will acknowledge your report, investigate thoroughly,
remove harmful content immediately if confirmed, and credit
responsible reporters (with their permission).
